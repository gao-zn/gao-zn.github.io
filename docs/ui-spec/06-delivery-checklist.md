# 06. UI 设计师交付自检清单

> **用途**: UI 设计师在交付每个组件设计规范之前，逐项自查，确保没有遗漏。  
> **使用方式**: 复制此清单到组件的 Figma 文件中，完成后逐项打勾 ✅

---

## 📋 通用检查项（每个组件必检）

### A. Design Token 引用检查

| 检查项 | 状态 |
|--------|------|
| 所有颜色值均引用了 `01-design-tokens.md` 中的 Token（未出现裸色值如 `#2575F5`） | ☐ |
| 所有字号引用了 Token 体系（未出现裸字号如 `14px`） | ☐ |
| 所有间距引用了 Token 体系（必须是 4px 的倍数） | ☐ |
| 圆角值引用了 Token 体系 | ☐ |
| 阴影值引用了 Token 体系 | ☐ |

### B. 状态覆盖检查

| 检查项 | 状态 |
|--------|------|
| Default（默认态）已给出 | ☐ |
| Hover（悬浮态）已给出 | ☐ |
| Active（按下态）已给出 | ☐ |
| Focus（聚焦态）已给出 | ☐ |
| Disabled（禁用态）已给出 | ☐ |
| Loading（加载态）已给出（如果组件有此状态） | ☐ |
| Error（错误态）已给出（如果组件有此状态） | ☐ |
| Empty（空数据态）已给出（如果组件展示数据） | ☐ |

### C. 双主题检查

| 检查项 | 状态 |
|--------|------|
| Light（亮色）主题所有状态已标注 | ☐ |
| Dark（暗色）主题所有状态已标注 | ☐ |
| 暗色主题下的对比度满足 WCAG AA（4.5:1 正文 / 3:1 大文本） | ☐ |
| 两个主题的截图已放入 Figma 或文档中 | ☐ |

### D. 边界情况检查

| 检查项 | 状态 |
|--------|------|
| 内容为空时如何展示？ | ☐ |
| 内容超出容器时如何截断？ | ☐ |
| 内容达到最大值时如何限制？ | ☐ |
| 内容为超长文本（如 50+ 字符）如何展示？ | ☐ |
| 最小/最大尺寸约束？ | ☐ |

### E. 布局关系检查

| 检查项 | 状态 |
|--------|------|
| 组件与上/下/左/右相邻组件的间距标注 | ☐ |
| 组件在页面中的典型位置标注 | ☐ |
| 组件在响应式不同断点下的行为标注 | ☐ |

## 📋 交付物文件清单

每个组件交付一个 Figma 页面 + 一份 Markdown 规范文档。

```
交付文件结构:
├── Figma 页面
│   ├── [组件名] - 结构拆解（标注各组成部分）
│   ├── [组件名] - 尺寸变体（small/middle/large）
│   ├── [组件名] - 状态矩阵 Light（每行一个状态）
│   ├── [组件名] - 状态矩阵 Dark（每行一个状态）
│   ├── [组件名] - 典型使用场景（1~3 个页面示例）
│   └── [组件名] - 边界情况（空/超长/极端）
│
└── Markdown 规范文档
    └── [组件名].md  （按 02-component-spec-template.md 模板填写）
```

| 检查项 | 状态 |
|--------|------|
| Figma 页面包含以上 6 个板块 | ☐ |
| Markdown 规范文档已按模板完整填写 | ☐ |
| 文档中所有色值均引用了 Token 变量名 | ☐ |
| 文档中附带了 Figma 页面链接 | ☐ |
| 已与前端开发评审确认（双方签字） | ☐ |

## 🔄 评审流程

```
UI 设计师完成设计
    │
    ▼
UI 自查本清单 → 修改不达标项
    │
    ▼
前端评审 → 标注技术可行性问题 → UI 修改
    │
    ▼
双方签字确认 → 设计稿锁定 → 前端开始开发
```
