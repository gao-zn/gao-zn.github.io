UI 设计规范交付文档

v1.0 · 2026-07-08

📋 规范体系总览

本规范定义了 DataCenter Platform 组件库的完整设计标准，UI 设计师和前端开发者需基于此文档协作。规范分三层递进，下层被上层引用，确保全局一致性。

🏗️ Design Tokens（原子层）

颜色、字号、间距、圆角、阴影等全局设计变量。
所有组件引用这一层，不直接写死数值。

对应文档：01-design-tokens.md

🧩 组件设计规范（分子层）

每个组件的结构、尺寸、状态矩阵、边界规则。
参照飞书设计规范的颗粒度，补齐状态覆盖。

参照：飞书 Breadcrumb 组件规范

🎯 全局交互规范（全局层）

动画系统、反馈机制、响应式断点、键盘交互。
跨组件统一的体验约定。

对应文档：05-interaction-spec.md

💡
设计工具适配：本规范兼容 Figma 和 MasterGo。
Figma 通过 Variables + Token Studio 插件导出；MasterGo 通过「样式导出为CSS」插件 + token.json 双向同步。详见 Design Tokens → 颜色系统。

13 UI 设计师交付自检清单

⚠️
UI 设计师在每个组件交付前，请逐项打勾自查。未通过自检的组件不要提交给前端评审。

通用检查项

  • 所有颜色值引用 Token（无裸色值如 #2575F5）
  • 所有字号引用 Token 体系（无裸字号）
  • 所有间距是 4px 的倍数，引用 Token
  • Default / Hover / Active / Focus / Disabled 五个状态已全部给出
  • Loading 态已覆盖（如果组件有加载场景）
  • Error 态已覆盖（如果组件有校验场景）
  • Empty 态已覆盖（如果组件展示数据）
  • Light（亮色）和 Dark（暗色）双主题已标注
  • 暗色主题对比度满足 WCAG AA（4.5:1 正文 / 3:1 大文本）
  • 内容为空 / 超长 / 极端情况已有处理方案
  • 与相邻组件的间距关系已标注
  • 组件在响应式断点下的行为已标注
