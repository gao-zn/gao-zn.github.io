# 升级框架中的过时插件

### 1️⃣ 图片压缩插件：`vite-plugin-imagemin` → 替代为 **`vite-plugin-image-optimizer`**

- **问题**：`vite-plugin-imagemin` 底层依赖 `imagemin`，而 `imagemin` 的某些二进制包下载困难，且项目已长期未维护。
- **替代方案**：[`vite-plugin-image-optimizer`](https://github.com/FatehAK/vite-plugin-image-optimizer)  
  - 基于 `sharp` 或 `squoosh`，安装简单（纯 JavaScript，无需下载额外二进制文件）。
  - 支持多种图片格式优化，配置灵活。
  - 示例配置：

    ```js
    import { ViteImageOptimizer } from 'vite-plugin-image-optimizer';
    export default {
      plugins: [ViteImageOptimizer()]
    }
    ```

---

### 2️⃣ 图标管理：`@purge-icons/generated` + `vite-plugin-purge-icons` → 替代为 **`unplugin-icons` + 按需集成**

- **问题**：PurgeIcons 项目已停止维护，且需要生成额外文件。
- **替代方案**：使用 [`unplugin-icons`](https://github.com/unplugin/unplugin-icons)  
  - 支持超过 100 个图标集（通过 Iconify），按需自动引入。
  - 与 Vite 无缝集成，无需手动生成文件。
  - 配合 `unplugin-auto-import` 可实现自动引入图标组件。
  - 示例配置：

    ```js
    import Icons from 'unplugin-icons/vite';
    export default {
      plugins: [Icons({ compiler: 'vue3' })]
    }
    ```

---

### 3️⃣ 样式按需加载：`vite-plugin-style-import` → 替代为 **`unplugin-vue-components` + `unplugin-auto-import`**

- **问题**：`vite-plugin-style-import` 需要手动配置样式导入规则，且对某些组件库支持不够智能。
- **替代方案**：组合使用 [`unplugin-vue-components`](https://github.com/unplugin/unplugin-vue-components) 和 [`unplugin-auto-import`](https://github.com/unplugin/unplugin-auto-import)  
  - 自动引入组件库（如 Ant Design Vue、Naive UI）并解析对应的样式。
  - 无需手动 import 组件，提升开发体验。
  - 示例配置（以 Ant Design Vue 为例）：

    ```js
    import Components from 'unplugin-vue-components/vite';
    import { AntDesignVueResolver } from 'unplugin-vue-components/resolvers';

    export default {
      plugins: [
        Components({
          resolvers: [AntDesignVueResolver({ importStyle: 'less' })]
        })
      ]
    }
    ```

---

### 4️⃣ HTML 注入/处理：`vite-plugin-html` → 替代为 **`vite-plugin-inject-html`** 或 **原生 `transformIndexHtml`**

- **问题**：`vite-plugin-html` 功能较全但略显臃肿，且可能与其他插件冲突。
- **替代方案**：  
  - 简单的变量替换可使用 Vite 原生 [`transformIndexHtml`](https://vitejs.dev/guide/api-plugin.html#transformindexhtml) 钩子。
  - 如果需要 ejs 模板功能，可使用 [`vite-plugin-inject-html`](https://github.com/alloc/vite-plugin-inject-html) 或 [`vite-plugin-html-config`](https://github.com/ahwgs/vite-plugin-html-config)。
  - 轻量且专注。

---

### 5️⃣ 构建压缩：`vite-plugin-compression` → 替代为 **`vite-plugin-compression2`**

- **问题**：原版 `vite-plugin-compression` 基于 `rollup-plugin-gzip`，功能稳定，但有新版本 `vite-plugin-compression2` 提供了更好的 TypeScript 支持和更现代的 API。
- **替代方案**：[`vite-plugin-compression2`](https://github.com/sxzz/vite-plugin-compression2)  
  - 完全兼容原版配置，使用更简单。
  - 支持 gzip、brotli 等算法。

---

### 6️⃣ 传统浏览器支持：`@vitejs/plugin-legacy` → **保持使用，但注意更新**

- **现状**：此插件仍由 Vite 团队维护，与 Vite 4 兼容，可继续使用。但若升级到 Vite 5+，需同步更新插件版本。
- 如果发现构建产物过大，可以考虑使用 `@vitejs/plugin-basic-ssl` 替代部分功能（但 legacy 主要提供 polyfill，无直接替代）。

---

### 7️⃣ 构建后脚本执行：`esno` → 替代为 **`tsx`** 或 **直接使用 `node --loader ts-node/esm`**

- **问题**：`esno` 是 `esbuild` 包装的 Node.js 运行器，但已不再积极维护。
- **替代方案**：  
  - **[`tsx`](https://github.com/privatenumber/tsx)**：基于 esbuild，支持 TypeScript 和 ESM，启动更快。  

    ```bash
    npm i -D tsx
    # 在 scripts 中改为： "postBuild": "tsx ./build/script/postBuild.js"
    ```

  - 或使用原生 Node.js 的 `--loader` 加载 `ts-node`（需配置）。
